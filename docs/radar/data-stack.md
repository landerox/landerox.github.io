# :material-database-cog: Data Engineering & High-Performance Stack

> The Modern Data Stack: separating storage from compute, prioritizing developer experience.

| Resource | Description |
|----------|-------------|
| [Apache Iceberg](https://iceberg.apache.org/){ target="_blank" } | Dominant open table format—adopted by Snowflake, AWS, and Databricks for lakehouse architectures |
| [Delta Lake](https://delta.io/){ target="_blank" } | Mature table format with deep Databricks integration and ACID transactions on data lakes |
| [Apache Hudi](https://hudi.apache.org/){ target="_blank" } | Best for upsert capabilities and incremental processing with streaming data sources |
| [Dagster](https://github.com/dagster-io/dagster){ target="_blank" } | My orchestration platform of choice—treats data assets as software products |
| [dbt](https://www.getdbt.com/){ target="_blank" } | The transformation standard—enables testing, versioning, and documentation in SQL pipelines |
| [Polars](https://pola.rs/){ target="_blank" } | Rust-powered DataFrames—order of magnitude faster than Pandas for large datasets |
| [DuckDB](https://duckdb.org/){ target="_blank" } | Embedded OLAP database—local analytics without infrastructure overhead |
| [Trino](https://trino.io/){ target="_blank" } | Distributed SQL query engine for interactive analytics across heterogeneous data sources |
| [Great Expectations](https://greatexpectations.io/){ target="_blank" } | Data quality framework for defining, monitoring, and validating data expectations |
| [MinIO](https://min.io/){ target="_blank" } | High-performance S3-compatible object storage for on-premise or hybrid cloud setups |
